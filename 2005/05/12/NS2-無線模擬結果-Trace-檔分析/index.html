NS2 無線模擬結果 Trace 檔分析

Posted on 2005-05-12 Edited on 2020-12-24 Disqus:

新 r 33.908773621 3 MAC — 334 mipv6_bu 70 [13a 1 4 800] ------- [4196353:0 0:0 32 4198400] 舊 s -t 1.000075000 -Hs 22 -Hd 2147483647 -Ni 22 -Nx 100.00 -Ny 2200.00 -Nz 0.00 -Ne -1.000000 -Nl MAC -Nw — -Ma 0 -Md ffffffff -Ms 14 -Mt 800 -Is 4222977.0 -Id 2147483647.0 -It ipv6_sol -Il 100 -If 0 -Ii 1 -Iv 255

欄位: event type (事件類型)

有四種基本事件類型

s: send (傳送) r: receive(接收) d:drop(丟棄) f: forward(轉送)<o:p></o:p>

<o:p>還有一種特別類型 m: move (移動) </o:p>

欄位2: general flag

-t: time (時間)

<o:p> </o:p>

欄位3: Next hop info (下一站的資訊)<o:p></o:p>

-Hs: id for this node<o:p></o:p>

-Hd: id for next hop towards the destination

<o:p> </o:p>

欄位4: Node property type tag (節點屬性類型標籤)<o:p></o:p>

-Ni: node id (節點ID) <o:p></o:p>

-Nx –Ny -Nz: node’s x/y/z coordinate (節點x/y/z的座標位置)<o:p></o:p>

-Ne: node energy level <o:p></o:p>

-Nl: trace level, such as AGT, RTR, MAC<o:p></o:p>

-Nw: reason for the event (事件發生原因)<o:p></o:p>

<o:p> </o:p>

欄位5: packet info at MAC level (封包在Mac層的資訊)<o:p></o:p>

-Ma: duration <o:p></o:p>

-Md: dest’s ethernet address<o:p></o:p>

-Ms: src’s ethernet address<o:p></o:p>

-Mt: ethernet type<o:p></o:p> <o:p></o:p>

欄位6: Packet information at IP level (封包在IP層的資訊)<o:p></o:p>

-Is: source address. Source port number (來源位置，a.b其中a為節點ID，b為埠號)<o:p></o:p>

-Id: dest address.dest port number (目的位置)<o:p></o:p>

-It: packet type (封包類型)<o:p></o:p>

-Il: packet size (封包大小)<o:p></o:p>

-If: flow id (資料流ID)<o:p></o:p>

-Ii: unique id (唯一的ID編號)<o:p></o:p>

-Iv: ttl value (Time To Live的值)

<o:p></o:p>

<o:p> </o:p> 欄位7: 封包在應用層的資訊。 包含的應用程式類型如 arp, tcp 或者是 adhoc 路由協定, 這個欄位都是以P所開頭的， 且標籤為隨著應用程式不同而不同

Trace 檔分析:

Packet Loss 數量計算: 不考慮Out-of-Order情況時, Packet Loss數量的計算方法最簡單, 每一個封包在傳送端發送前，傳送端都會給封包一個序號，序號是連續性的，因此若是在接收到發現序號有不連續的發生，則可視為有封包的移失。

OWD&IPDV計算: 最重要的欄位是 Sending Time(傳送時間)和Sequence Number(封包序號), 可以用來計算出One Way Delay (OWD)和IP Delay Variance (IPDV，或是Jitter)。

One Way Delay = 接收時間 – 傳送時間<o:p></o:p> **IPDV = |目前量測到的OWD – 上一次量測到的OWD |
